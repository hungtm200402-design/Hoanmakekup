# Context Map

This repo uses a multi-context domain documentation layout.

## Contexts

| Context | File | Purpose |
| ------- | ---- | ------- |
| Frontend | `docs/context/frontend.md` | Customer-facing booking, browsing, shopping, account, and checkout experiences. |
| Backend/Admin | `docs/context/backend.md` | Administrative workflows for appointments, services, products, orders, customers, and trusted product indexing. |

## Shared Decisions

System-wide architectural decisions live in `docs/adr/`.

When work spans both customer-facing and admin workflows, read both context files before proposing changes.
